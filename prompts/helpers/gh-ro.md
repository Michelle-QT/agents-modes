`gh-ro` is read-only GitHub CLI pinned to `github.com`: the only no-ask way to read the forge (pull requests, issues, CI, comments). Use it for review, e.g. `gh-ro pr diff 192`, `gh-ro pr view 192 --comments`, `gh-ro pr checks 192`. It forwards only read subcommands to `gh`, refuses all writes, rejects alternate forge hosts, and authenticates through the standard `gh` config/keychain without forwarding inherited token environment variables. A plain `gh` is gated, so use `gh-ro`, never bare `gh`.
